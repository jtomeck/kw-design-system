--- title: Empty States layout: collections # Page Hero definition: Moments in a user's experience where there is no content or data to display. excerpt: Empty states provide us with the opportunity to improve the user's experience by helping to avoid confusion and keep them engaged. A useful empty state should provide the user with information about what is happening, why it's happening and what they should do about it. # Page Content sections: - heading: When should empty states be utilized? list: - title: First-time use of app/feature content: It is helpful to provide information to help users understand what the application/tool/feature is and what value it provides. Users should then be provided with instructions and/or actions to get them started. image: https://fakeimg.pl/1024x600/?retina=1&text=Screenshot&font=noto - title: An action is required to populate content content: Sometimes a user must complete an action in order to populate data and/or content within a section of the application. It is helpful to provide content that makes them aware of this, as well as instructions on what they need to do to populate it. image: https://fakeimg.pl/1024x600/?retina=1&text=Screenshot&font=noto - title: An action cleared the content content: Sometimes a user action can clear data and/or content that was there previously. It is helpful to let the user know that it is okay that there is nothing there. In some cases it may be helpful to tell the user what action cleared the section or even give them the option to undo that action. image: https://fakeimg.pl/1024x600/?retina=1&text=Screenshot&font=noto - title: There are no results to display content: Sometimes a user will complete an action that does not yield any results. It is helpful to prompt the user, letting them know their action did not return anything and to perform it again with modifications. image: https://fakeimg.pl/1024x600/?retina=1&text=Screenshot&font=noto ---
